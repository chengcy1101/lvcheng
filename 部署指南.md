# 绿城AI案场销助 - 部署指南

本项目包含前端（静态HTML/CSS/JS）和后端（Node.js + Express）两部分。以下是几种不同的部署方案。

---

## 方案一：Vercel（推荐，免费且简单）

Vercel 是最适合 Next.js 和 Node.js 项目的免费部署平台，支持自动部署和 HTTPS。

### 步骤：

1. **创建 GitHub 仓库**
   - 在 GitHub 上创建一个新仓库
   - 将您的代码推送到 GitHub

2. **配置 Vercel**
   - 访问 https://vercel.com
   - 使用 GitHub 账号登录
   - 点击 "New Project"
   - 导入您的 GitHub 仓库

3. **修改部署配置**

   在项目根目录创建 `vercel.json`：
   ```json
   {
     "version": 2,
     "builds": [
       {
         "src": "backend/server.js",
         "use": "@vercel/node"
       }
     ],
     "routes": [
       {
         "src": "/api/(.*)",
         "dest": "/backend/server.js"
       },
       {
         "src": "/(.*)",
         "dest": "/frontend/$1"
       }
     ]
   }
   ```

4. **设置环境变量**
   - 在 Vercel 项目设置中添加环境变量
   - 添加 `API_KEY`、`ENDPOINT`、`MODEL` 等变量

5. **修改前端 API 调用地址**

   更新 `frontend/script.js` 中的 API 地址：
   ```javascript
   // 从硬编码的 localhost 改为相对路径
   const response = await fetch('/api/chat', { ... });
   ```

6. **部署**
   - 提交代码到 GitHub
   - Vercel 会自动部署
   - 完成后您会获得一个类似 `your-project.vercel.app` 的域名

## 方案二：Netlify + Render（前后端分离）

### 前端部署到 Netlify（免费）

1. **将前端文件夹推送到 GitHub**
2. **访问 https://www.netlify.com**
3. **导入 GitHub 仓库**
4. **设置部署配置：**
   - Publish directory: `frontend`
   - Build command: 留空（不需要构建）

### 后端部署到 Render（免费）

1. **访问 https://render.com**
2. **创建一个新的 Web Service**
3. **连接 GitHub 仓库**
4. **配置：**
   - Runtime: Node
   - Build Command: `npm install`
   - Start Command: `npm start`
   - Root Directory: `backend`

5. **添加环境变量**

6. **修改前端 API 地址**
   ```javascript
   const response = await fetch('https://your-backend.onrender.com/api/chat', { ... });
   ```

## 方案三：阿里云/腾讯云服务器（更灵活，需要付费）

如果您想拥有完全控制的服务器：

### 步骤：

1. **购买云服务器**
   - 阿里云：https://www.aliyun.com
   - 腾讯云：https://cloud.tencent.com
   - 建议配置：2核4GB，Ubuntu 20.04

2. **连接服务器**
   ```bash
   ssh root@your-server-ip
   ```

3. **安装 Node.js 和 Nginx**
   ```bash
   curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
   sudo apt-get install -y nodejs nginx
   ```

4. **上传代码到服务器**
   ```bash
   scp -r /Users/chengcc/Desktop/lvcheng root@your-server-ip:/var/www/
   ```

5. **在服务器上安装依赖并启动后端**
   ```bash
   cd /var/www/lvcheng/backend
   npm install
   npm start
   ```

6. **配置 Nginx**

   创建 `/etc/nginx/sites-available/lvcheng`：
   ```nginx
   server {
       listen 80;
       server_name your-domain.com;
       
       location / {
           root /var/www/lvcheng/frontend;
           index index.html;
       }
       
       location /api/ {
           proxy_pass http://localhost:3001;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_cache_bypass $http_upgrade;
       }
   }
   ```

   启用配置：
   ```bash
   sudo ln -s /etc/nginx/sites-available/lvcheng /etc/nginx/sites-enabled/
   sudo nginx -t
   sudo systemctl restart nginx
   ```

7. **配置 HTTPS（使用 Let's Encrypt）**
   ```bash
   sudo apt-get install certbot python3-certbot-nginx
   sudo certbot --nginx -d your-domain.com
   ```

8. **使用 PM2 管理后端进程**
   ```bash
   npm install -g pm2
   pm2 start server.js --name lvcheng-backend
   pm2 startup
   pm2 save
   ```

## 方案四：使用 Docker（适合专业用户）

### 创建 Dockerfile

在 `backend/` 目录创建 `Dockerfile`：
```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install --only=production
COPY . .
EXPOSE 3001
CMD ["npm", "start"]
```

### 使用 Docker Compose

在项目根目录创建 `docker-compose.yml`：
```yaml
version: '3'
services:
  backend:
    build: ./backend
    ports:
      - "3001:3001"
    env_file:
      - ./backend/.env
    restart: unless-stopped

  frontend:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./frontend:/usr/share/nginx/html
    depends_on:
      - backend
```

## 推荐选择

- **想快速免费体验**：选择 Vercel（方案一）
- **完全分离且免费**：选择 Netlify + Render（方案二）
- **需要完全控制**：选择云服务器（方案三）
- **熟悉容器化**：选择 Docker（方案四）

## 重要提示

1. **API 密钥安全**：永远不要将 `.env` 文件提交到公开的 GitHub 仓库
2. **CORS 配置**：生产环境需要更新 CORS 设置，允许您的域名
3. **监控**：设置日志和监控，确保服务稳定运行
4. **备份**：定期备份重要数据和配置
